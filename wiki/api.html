Utils

The Utils namespace provides utility functions for encoding, decoding, and serialization of data. These functions are particularly useful for handling encoded data structures and converting values to string representations.

Utils.decode()

decode(dec: any): any

Recursively decodes encoded values in an object structure. Detects objects with __encoding_type__ and __val__ properties and decodes them. Currently supports base64 encoding type.

Parameters

  • dec any - The value to decode (can be any type)

Returns

any - The decoded value with all nested encoded values converted

Example

// Decodes a base64 encoded string
const decoded = Utils.decode({ 
    __encoding_type__: "base64", 
    __val__: [72, 101, 108, 108, 111] 
});
await Log.debug(decoded); // "Hello"

// Recursively decodes nested objects
const obj = Utils.decode({ 
    name: { __encoding_type__: "base64", __val__: [74, 111, 104, 110] } 
});
await Log.debug(obj); // { name: "John" }

Utils.encode()

encode(enc: any, options?: { string: "base64" }): any

Recursively encodes values in an object structure. Converts strings to an encoded format with __encoding_type__ and __val__ properties. Arrays and objects are processed recursively.

Parameters

  • enc any - The value to encode (can be any type)
  • options { string: "base64" } - Encoding options = { string: "base64" }

Returns

any - The encoded value with all nested strings converted to encoded format

Example

// Encodes a string to base64 format
const encoded = Utils.encode("Hello");
await Log.debug(encoded); 
// { __encoding_type__: "base64", __val__: [72, 101, 108, 108, 111] }

// Recursively encodes nested objects
const obj = Utils.encode({ name: "John", age: 30 });
await Log.debug(obj); 
// { name: { __encoding_type__: "base64", __val__: [...] }, age: 30 }

Utils.serialize()

serialize(obj: any): string

Serializes a value to a string representation. If the value is already a string, returns it unchanged. Otherwise, converts the value to a JSON string.

Parameters

  • obj any - The value to serialize

Returns

string - The string representation of the value

Example

Utils.serialize("hello"); // Returns: "hello"
Utils.serialize({ key: "value" }); // Returns: '{"key":"value"}'
Utils.serialize(123); // Returns: "123"
Utils.serialize([1, 2, 3]); // Returns: "[1,2,3]"

Callbacks

The Callbacks namespace provides functions for handling asynchronous events and messages. These functions are particularly useful for managing communication between different parts of the application.

window.renweb.onReady

window.renweb.onReady = async () => { ... }

Called once the application is fully initialised and the window is ready for interaction. Assign this callback before or during window.onload — if it is assigned after the ready signal has already fired (e.g. deep inside an async window.onload after several awaits), the framework detects this and calls it immediately.

Returns

void | Promise<void> - May be async

Example

window.renweb.onReady = async () => {
    await Window.show(true);
};

window.renweb.onTerminate

window.renweb.onTerminate = async () => { ... }

Called just before the window closes, whether triggered programmatically via Window.terminate(), by the user clicking the title-bar X button, or by Ctrl+C in the terminal. Use this to save state, flush logs, or perform any other cleanup before exit.

Note: When triggered by Window.terminate() the callback is fully await-ed before the window closes. When triggered by the OS close button or Ctrl+C the callback is best-effort — very long async operations may be cut short.

Returns

void | Promise<void> - May be async

Example

window.renweb.onTerminate = async () => {
    await Config.saveConfig();
};
